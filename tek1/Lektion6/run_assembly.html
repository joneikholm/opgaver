Hello World — Running Assembly in the Terminal

x86-64 · Ubuntu · NASM
1
Write
You write the source code in a plain text file — here named hello.asm (the .asm extension is the convention for assembly files). Using nano (or any editor), you type assembly mnemonics like mov and syscall — human-readable symbols that each map to a single CPU instruction. This is the only step written and read by a human.
$ nano hello.asm
2
Assemble
NASM (Netwide Assembler) translates each mnemonic into binary machine code. -f elf64 sets the output format: ELF (Executable and Linkable Format) is the standard binary format on Linux. Other formats exist — elf32 for 32-bit Linux, win64 for Windows (PE format), macho64 for macOS, bin for raw binary used in bootloaders. -o hello.o names the output object file.
$ nasm -f elf64 hello.asm -o hello.o
3
Link
ld — short for link editor, the traditional Unix name for the linker — takes hello.o and produces the final executable. -o hello sets the output filename. It resolves all labels to real memory addresses, locates the _start entry point, and writes a complete ELF header — the metadata Linux needs to load and run the program.
$ ld -o hello hello.o
4
Run
./hello tells the shell to run the file named hello in the current directory (./). The OS reads the ELF header, loads the binary into memory, and jumps to _start. The CPU executes each instruction directly — no interpreter, no virtual machine. syscall briefly hands control to the Linux kernel to print to the screen and exit.
$ ./hello
Hello world!
