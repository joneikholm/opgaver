User space vs Kernel space

De fleste CPU-instruktioner (fx mov, add, mul) kører direkte på hardwaren i user mode. Men adgang til delte ressourcer (filer, tastatur, netværk, GPU, proceskontrol) skal gå via kernen gennem et systemkald (syscall).

Hvad user space kan (hurtigt, direkte CPU-udførelse)

  • Regning, løkker, branches, databehandling
  • Arbejde med egne memory pages (læse/skrive hvor det er tilladt)
  • Bygge buffers: tekst, billeder, lyd, netværksdata

Hvad kræver kernel space (beskyttet, delt, følsomt)

  • Enheder: skærm, disk, netkort, GPU-driver
  • Processer: start/stop, scheduling, signals
  • Isolation: “du må ikke røre andre processers memory”
Kerneidé: Beregn i user space. Hardware og systemressourcer går gennem kernen.

Hello world i assembly

Programmet skriver ved at kalde write(fd, buf, len) og afslutter med exit(code). Det bruger ikke C-biblioteker — kun kernens interface.

Kode

section .data
  msg db "Hello world!", 0ah

section .text
  global _start

_start:
  mov rax, 1      ; syscall #1 = write
  mov rdi, 1      ; fd 1 = stdout
  mov rsi, msg    ; pointer til bytes
  mov rdx, 13     ; antal bytes
  syscall

  mov rax, 60     ; syscall #60 = exit
  mov rdi, 0      ; exit-kode 0
  syscall

Registrer for syscalls (x86-64 Linux)

  • rax = syscall-nummer
  • rdi = arg1
  • rsi = arg2
  • rdx = arg3
write(1, msg, 13) betyder: “send 13 bytes fra adressen msg til destinationen fd=1 (stdout).”

Hvis du ændrer rdx til 12, skriver du 12 bytes (ingen newline). Hvis du fjerner exit, “falder” CPU’en ud af koden → crash (ofte SIGSEGV / -11).

stdout (fd 1) er et stik

Fil-deskriptorer er små tal, der peger ind i en tabel for processen (holdt af kernen). Som standard:

Standard fil-deskriptorer

  • 0 = stdin (input)
  • 1 = stdout (normal output)
  • 2 = stderr (fejloutput)
Programmet ved ikke, hvad stdout er forbundet til. Det skriver bare bytes til fd 1.

Hvorfor det “automatisk” vises i terminalen

  • Shell’en har allerede fd 0/1/2 koblet til en terminal-enhed.
  • Når shell’en starter dit program, arver programmet forbindelserne.
  • Derfor peger fd 1 typisk på samme terminal som shell’en.
# Samme program, forskellige destinationer
./hello            # stdout → terminalvindue
./hello > out.txt  # stdout → fil
./hello | wc -c    # stdout → pipe → andet program

Hvorfor vi har kernel space

Hvis user-programmer kunne styre enheder eller vilkårlig memory direkte, kunne én fejl (eller malware) crashe eller overtage hele maskinen. CPU’en håndhæver derfor en grænse: user mode vs kernel mode.

Hardware-sikkerhed (håndhævet af CPU)

  • Virtual memory: hver proces får sit eget address space
  • Page permissions: user pages vs supervisor-only pages
  • Privilegerede instruktioner: blokeret i user mode
Hvis et user-program rører memory det ikke ejer → CPU trapper → kernen stopper processen (fx SIGSEGV).

Kernens opgave

  • Koordinere delt hardware (disk, netværk, GPU)
  • Håndhæve rettigheder og isolation
  • Levere sikre services via syscalls
  • Scheduling: hvem kører hvornår, hvem får enhederne

Et syscall er ikke “din kode med flere rettigheder”. Det er CPU’en der skifter til kernen, så kernel-kode kører på dine vegne.

Hvem tegner faktisk “Hello world”?

Dit program skriver bytes til stdout. Kernen ruter bytes til en terminal-enhed. En terminal-applikation (user space) renderer bogstaver med en font og beder grafikløsningen om at vise dem. GPU’en ender med at male pixels.

Pipeline

Dit program user space
write(1, msg, len) syscall-grænse
Kernen ruter bytes → pseudo-terminal kernel space
Terminal læser bytes, tegner glyphs user space
Window system / driver sender arbejde kernel + drivere
GPU + display controller hardware
Vigtigt: Kernen renderer normalt ikke bogstaverne. Den koordinerer og beskytter.

Én sidste “wow”-fakta

  • stdout er en abstraktion — kan være terminal, fil, pipe eller netværk.
  • GUI-apps laver ikke syscall pr. pixel — de bygger buffers/kommandoer og afleverer dem i store bidder.
  • Samme princip: beregn i user space, brug syscalls til delte ressourcer.
I én sætning:
User-programmer beregner frit,
men må spørge kernen for sikkert at bruge delt hardware.
